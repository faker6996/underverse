1 UEditor là gì

UEditor là rich-text editor dựa trên TipTap/ProseMirror. Người dùng có thể viết nội dung, định dạng chữ, chèn bảng, ảnh, liên kết, emoji, code block, callout, bookmark và tệp đính kèm ngay trong ứng dụng web.

UEditor phù hợp cho các màn hình cần soạn thảo nội dung dài như bài viết, mô tả sản phẩm, ghi chú nội bộ, tài liệu hướng dẫn, biên bản họp hoặc nội dung có bảng tính nhẹ. Người dùng không cần biết HTML; mọi thao tác chính đều có thể làm bằng toolbar, menu nổi hoặc slash command.

Component có ba kiểu hiển thị chính:

  • default: có toolbar đầy đủ phía trên editor.
  • minimal: toolbar rất gọn, hiện chỉ có Bold, Italic và Bullet list.
  • notion: biến thể hiển thị nhẹ hơn cho trải nghiệm giống Notion; toolbar, menu bar và bubble menu vẫn do các prop riêng quyết định.

3 Toolbar và định dạng văn bản

Toolbar là nơi dùng nhanh các lệnh định dạng phổ biến. Một số nút là nút bấm trực tiếp, một số nút mở dropdown hoặc palette.

Trong variant="minimal", toolbar không phải bản rút gọn của toàn bộ toolbar đầy đủ; code hiện chỉ render ba nút Bold, Italic và Bullet list. Nếu tài liệu end-user cần nhắc đến font, ảnh hoặc bảng thì nên dùng default/notion với showToolbar phù hợp.

Cách định dạng một đoạn text

  1. Bôi đen đoạn text cần chỉnh hoặc đặt con trỏ tại vị trí muốn nhập tiếp.
  2. Chọn nhóm định dạng trên toolbar: bold, italic, underline, màu chữ, highlight hoặc typography.
  3. Nếu chọn dropdown, bấm vào lựa chọn mong muốn. Editor áp dụng ngay cho vùng chọn.
  4. Nếu muốn bỏ định dạng, chọn lại nút đang active hoặc dùng Clear Formatting trong menu bar.

showFloatingMenu đang có trong type và CustomFloatingMenu đã được định nghĩa trong code, nhưng UEditor.tsx hiện chưa render component này. Trong bản hiện tại, hãy hướng dẫn người dùng dùng slash command bằng phím /; chỉ mô tả nút floating add-block nếu đội triển khai nối lại prop này.

Cách dùng slash command

  1. Đặt con trỏ ở dòng trống hoặc vị trí muốn chèn block.
  2. Gõ /. Danh sách Basic Blocks sẽ mở ngay dưới con trỏ.
  3. Gõ tiếp từ khóa như table, quote, file để lọc danh sách.
  4. Dùng chuột để chọn item, hoặc dùng ↑/↓ rồi nhấn Enter.
  5. Nhấn Esc hoặc click ra ngoài để đóng menu.
Bộ lọc slash command hiện lọc theo title của command, không lọc theo phần mô tả. Ví dụ gõ file ra File Attachment, nhưng gõ theo một từ chỉ nằm trong description có thể không ra kết quả.

Emoji autocomplete

Ngoài emoji picker trên toolbar, UEditor hiện bật EmojiSuggestion. Khi gõ :, editor mở lưới emoji gợi ý; người dùng có thể gõ tiếp tên emoji để lọc, dùng phím mũi tên để di chuyển và nhấn Enter để chèn.

Emoji suggestion hiển thị tối đa 64 emoji trong popup. Khi không có query, danh sách dùng các emoji đầu trong bộ EMOJI_LIST.

5 Bubble menu khi chọn nội dung

Khi bôi đen text hoặc chọn một node đặc biệt, bubble menu nổi phía trên vùng chọn. Menu này giúp chỉnh nhanh mà không cần di chuyển lên toolbar.

Ví dụ: đoạn text đang được chọn có thể đổi màu, tạo link, bật bold/italic hoặc mở panel typography.

Bubble menu có thêm nút công thức và màu nền/border khi selection nằm trong bảng.

Với link, bubble menu chuyển sang chế độ preview: hiển thị URL, nút mở tab mới, sửa link và gỡ link. Với ảnh, bubble menu hiển thị layout block/left/right, kích thước S/M/L, reset size và xóa ảnh.

Bubble menu có độ trễ hiển thị ngắn để tránh nhấp nháy khi selection thay đổi. Khi người dùng mở input link hoặc formula, menu được giữ mở để không mất focus giữa chừng.

6 Liên kết, ảnh, bookmark và tệp đính kèm

Liên kết

Chọn text rồi bấm nút link để nhập URL. UEditor chỉ cho phép protocol an toàn như http, https, mailto, tel hoặc relative URL; URL nguy hiểm như javascript: và protocol-relative URL dạng //example.com bị từ chối.

  1. Bôi đen text muốn gắn link.
  2. Bấm biểu tượng link trên toolbar hoặc bubble menu.
  3. Dán URL vào input. Nếu URL không có protocol nhưng hợp lệ, editor cố gắng chuẩn hóa thành https://....
  4. Bấm xác nhận. Text sẽ có style link và có thể preview lại khi đặt con trỏ/chọn link.

Ảnh

Ảnh có thể chèn từ URL, upload từ máy, paste hoặc kéo thả. Mặc định ảnh được giữ dạng base64 khi đang soạn thảo; khi lưu có thể gọi prepareContentForSave() để upload base64 thành URL thật.

Image

Wrapped image: chọn ảnh rồi dùng menu ảnh để chuyển giữa Image Block, Wrap Text Right và Wrap Text Left. Có thể chọn width Small, Medium, Large, reset size hoặc xóa ảnh.

Trên mobile, ảnh wrap được đưa về block để tránh text bị bó hẹp.

Ảnh hiện có layout block/trái/phải và preset kích thước S/M/L, không chỉ là chèn ảnh đơn giản.

Chỉnh ảnh sau khi chèn

  1. Click vào ảnh để chọn node ảnh. Ảnh sẽ có viền/ring chọn.
  2. Dùng bubble menu hoặc dropdown Image trên toolbar.
  3. Chọn Image Block để ảnh đứng riêng một dòng, Wrap Text Right để ảnh nằm trái và chữ chảy bên phải, hoặc Wrap Text Left để ảnh nằm phải và chữ chảy bên trái.
  4. Chọn S/M/L để đổi preset width. Dùng reset để bỏ kích thước custom, hoặc delete để xóa ảnh.

Ảnh còn có handle resize ở góc dưới phải. Khi kéo, editor giữ tỷ lệ ảnh, giới hạn tối thiểu khoảng 40px và không cho vượt quá vùng editor. Width/height được lưu vào node; reset size sẽ xóa kích thước custom và preset size. Preset hiện tại: ảnh block S/M/L khoảng 180/280/380px; ảnh wrap trái/phải S/M/L khoảng 140/200/260px.

Bookmark và file card

Slash command có Bookmark Card để nhúng link dạng thẻ preview. Lệnh bookmark hiện dùng prompt nhập URL đơn giản; sau khi chèn, nếu truyền fetchMetadata, bookmark có thể hiển thị title, description, image hoặc publisher. Nếu paste một URL http/https thuần vào đoạn paragraph đang trống, extension bookmark có thể tự chuyển đoạn đó thành bookmark card.

File Attachment mở file picker, tạo thẻ tệp bằng data URL. File card hiển thị icon theo MIME phổ biến như PDF, image, audio, video hoặc generic, hiển thị tên file/kích thước, có nút download và trạng thái upload/retry. Upload ngay sau khi chèn dùng thứ tự handler trong code: uploadFile trước, sau đó có thể fallback qua upload save-time wrapper hoặc uploadImage tùy cấu hình hiện có.

Bookmark có fallback title theo hostname/URL nếu không lấy được metadata. Nếu fetch metadata lỗi, card có trạng thái lỗi và nút retry trong preview.
Bookmark và file card là node riêng, không phải link/ảnh thường. Khi render lại HTML đã lưu, cần giữ nguyên các thuộc tính data cần thiết để UEditor đọc lại đúng node.

7 Bảng: tạo, chỉnh sửa, resize, reorder và menu điều khiển

Có ba cách tạo bảng:

  1. Bấm nút Table trên toolbar và rê chuột trên grid 8×8 để chọn số hàng/cột.
  2. Gõ /table để chèn nhanh bảng 3×3 có header row.
  3. Dùng menu bar: Insert > Table hoặc Table > Insert Table.

Quy trình làm việc với bảng

  1. Chèn bảng bằng toolbar, menu bar hoặc slash command.
  2. Click vào ô để nhập nội dung. Dùng Tab để đi sang ô kế tiếp nếu trình duyệt/editor cho phép.
  3. Hover cạnh phải cột để hiện resize handle, kéo ngang để đổi chiều rộng cột.
  4. Hover cạnh dưới hàng để hiện row resize, kéo dọc để đổi chiều cao hàng.
  5. Hover bên phải hoặc phía dưới bảng để dùng rail + thêm nhanh cột/hàng; kéo rail ra xa hơn để thêm nhiều cột/hàng trong một lần.
  6. Kéo handle hàng/cột để đổi thứ tự nếu cần sắp xếp lại bảng.
  7. Chọn nhiều ô để merge; chọn ô đã merge để split.
  8. Dùng bubble menu để đổi màu nền, border hoặc nhập công thức cho ô.

Code cố gắng giữ background, border, colspan/rowspan, width cột, height hàng, formula/computed value, number format và một số inline mark như bold/italic/underline/color/highlight; tuy vậy sau khi paste vẫn nên kiểm tra lại layout, header và công thức vì dữ liệu ngoài có thể không có đầy đủ thuộc tính nội bộ của UEditor.
Bubble menu trong bảng có thêm màu nền ô, border ô, công thức và merge/split cell.

8 Công thức trong bảng

UEditor hỗ trợ công thức bảng ở mức lightweight spreadsheet. Có thể nhập công thức trực tiếp trong ô hoặc dùng nút Σ Formula trong bubble menu khi con trỏ ở trong bảng.

ABC
1020=SUM(A1:B1)
51520
Formula panel: =SUM(A1:B2) · Format: Text / 123 / $ / % / Date · Apply · Clear · Recalculate
Công thức dùng địa chỉ kiểu spreadsheet: A1, B2, vùng A1:B3 và các hàm SUM/AVG/MIN/MAX/COUNT.

Cách nhập công thức bằng ô bảng

  1. Click vào ô muốn hiển thị kết quả.
  2. Gõ dấu = rồi nhập công thức, ví dụ =SUM(A1:B3).
  3. Khi đang nhập hàm, có thể dùng gợi ý formula nếu dropdown xuất hiện.
  4. Rời ô, đổi selection hoặc blur editor để công thức được promote và tính lại.
  5. Nếu muốn ép tính lại, mở Formula panel và bấm Recalculate.

Cách nhập công thức bằng Formula panel

  1. Click vào ô bảng cần đặt công thức.
  2. Chọn nút Σ trên bubble menu.
  3. Nhập công thức trong input. Có thể nhập có hoặc không có dấu =; editor sẽ tự chuẩn hóa thành công thức có =.
  4. Chọn định dạng hiển thị: Text, Number, Currency, Percent hoặc Date.
  5. Bấm Apply để áp dụng, Clear để xóa công thức, Recalculate để tính lại bảng đang chọn.

10 Lưu, xuất và chế độ đọc

Lưu nội dung

Thông thường app lấy HTML qua onChange/onHtmlChange. Nếu nội dung có ảnh base64 hoặc file card inline, nên gọi prepareContentForSave() qua ref trước khi lưu.

const prepared = await editorRef.current?.prepareContentForSave({ throwOnError: true })
await api.savePost({ content: prepared?.html ?? "" })

Kết quả trả về gồm HTML đã chuẩn hóa, danh sách item đã upload, inline image URLs và lỗi upload nếu có. uploadImageConcurrency mặc định là 3 và được dùng để giới hạn upload song song khi chuẩn bị lưu.

Luồng lưu khuyến nghị

  1. Người dùng soạn nội dung bình thường; ảnh paste/upload có thể đang là base64.
  2. Khi bấm Save, gọi prepareContentForSave({ throwOnError: true }).
  3. Hàm tìm ảnh base64 và file card data URL cần upload rồi chạy upload với giới hạn concurrency.
  4. Nếu upload thành công, HTML trả về đã thay data URL bằng URL thật.
  5. Lưu HTML đã chuẩn hóa vào database hoặc API backend.
  6. Nếu có lỗi và throwOnError=true, bắt exception để báo người dùng lưu thất bại hoặc yêu cầu thử lại.
Field trong kết quảÝ nghĩa
htmlHTML cuối cùng nên dùng để lưu.
uploadedDanh sách ảnh/file card đã upload thành công cùng URL/meta.
inlineImageUrlsURL ảnh inline đã được phát hiện trong nội dung.
inlineUploadedDanh sách ảnh inline base64 đã upload, kèm index và thông tin file nếu có.
errorsLỗi upload theo index. Nếu không bật throw, app tự xử lý danh sách này.

Handler upload khi lưu có thể trả về URL string hoặc object có dạng { url, ...meta }. Ảnh base64 cần uploadImageForSave. File card data URL dùng uploadFileForSave nếu có, nếu không fallback sang uploadImageForSave.

Xuất HTML

Khi dùng menu bar, File > Export HTML sẽ gọi onExport nếu app truyền callback. Nếu không có callback, editor tạo một Blob HTML đơn giản và tải về file document.html. Đây là export nhanh cho người dùng, không thay thế logic lưu chính thức của app.

Chế độ đọc

Khi editable=false, người dùng không sửa được nội dung; toolbar và resize table không còn tác dụng. Link trong read-only có thể mở tab mới khi click nếu selection đang rỗng.

Read-only vẫn dùng UEditor để render HTML nên các node như table, image layout, bookmark, file card có cơ hội hiển thị nhất quán với editor. Nếu render HTML bằng component khác, cần tự đảm bảo CSS và sanitize phù hợp.
